- `hard_reset` via WebSocket: desregistra Service Worker, borra caches, limpia localStorage, recarga player
- Service Worker cache bump v1→v2 con limpieza automática de caches viejas
- Version bump player 1.0.0→1.1.0 para auto-actualización en TVs existentes
- Purga de Redis cache de player al iniciar servidor (evita datos stale sin `purpose`)
- Range Request support (206 Partial Content) en `publicMediaRoutes` para streaming de video en Smart TVs
- Fix: `itemCount = items.length` sin declaración `var` en strict mode (bloqueaba reproducción)
- Fix: `video.play().catch()` para Smart TVs que no soportan Promises
- Video error handler: muestra título + "Video no disponible" y salta al siguiente tras 5s

## Etapa 5 — Player TV Fixes (Completada)
- Video stall detection: timer verifica cada 2s que currentTime avanza, después de 3 stalls consecutivos salta al siguiente item
- WebSocket authentication: token JWT enviado como query parameter `?token=`, validado en server contra screenId
- Server-side ping/pong: intervalo de 30s envía ping para detectar conexiones muertas, cleanup automático en close
- Exponential backoff: reconnect delay incrementa de 5s → 10s → 20s → 40s → 60s (máx), resetea a 5s en conexión exitosa
- Service Worker v9: estrategia stale-while-revalidate para videos (sirve cache inmediatamente, actualiza en background)
- Cache-first mantenido para imágenes (cambian menos frecuentemente)
- Version bump SW v8→v9 para auto-actualización en TVs existentes

## Etapa 6 — Storage Hardening (Completada)
- Monitoreo de espacio en disco: `getDiskUsage()` usa `fs.statfs()` para obtener used/available/total
- Tamaño de uploads: `getUploadDirSize()` calcula espacio total usado en directorio de uploads
- Listado de archivos: `listAllFiles()` retorna todos los archivos con orgId, filename y size
- Endpoint `GET /api/media/stats`: estadísticas de disco y uploads por organización (solo admin)
- Endpoint `GET /api/media/orphans`: detecta archivos en disco sin registro en DB (solo admin)
- Endpoint `POST /api/media/cleanup-orphans`: elimina archivos huérfanos del disco (solo admin)
- MediaLibrary.tsx: dashboard con 3 cards (disco usado, archivos en org, espacio disponible)
- Botón de limpieza de huérfanos con confirmación y contador de archivos
- Bind mount configurable: `UPLOAD_DIR` env var ya soporta paths absolutos para NAS/disco externo
- Cuotas por organización: stats endpoint muestra uso por org vs total del sistema

## Etapa 7 — Cloudflare R2 Storage Provider (Completada)
- Instalación de `@aws-sdk/client-s3` y `@aws-sdk/s3-request-presigner` para compatibilidad S3
- Interfaz `StorageProvider` con métodos: saveFile, getFileStream, deleteFile, getFileSize, getContentType, getFileUrl
- `LocalStorageProvider`: refactor del código original, mantiene compatibilidad con filesystem local
- `R2StorageProvider`: implementación completa usando Cloudflare R2 (S3-compatible API)
- Factory function `getStorage()`: selecciona provider basado en `STORAGE_PROVIDER` env var (local|r2)
- Nuevas env vars: `STORAGE_PROVIDER`, `R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_BUCKET_NAME`, `R2_PUBLIC_URL`
- URLs configurables: R2 soporta `R2_PUBLIC_URL` para bucket público con CDN, fallback a proxy via API
- Range requests: R2 provider soporta `Range` header para streaming de video parcial
- Compatibilidad backward: media routes funcionan sin cambios, solo llaman a `getStorage()`
- Validación de config: error claro si R2 provider seleccionado sin credenciales completas
